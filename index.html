⚙️ Repository Configuration

Connect once — all uploaded files (test reports, bill scans, prescriptions) will be pushed to a private GitHub repository you own. Files are stored safely and accessible from any device.

1
Create a private GitHub repository
Go to github.com/new → name it e.g. kanika-pregnancy-docs → set Private → Create.
2
Generate a Personal Access Token
Go to GitHub → Settings → Developer settings → Tokens (classic) → tick repo scope → copy it.
3
Paste details below & test connection
Your token is stored only in this browser's localStorage — it never leaves your device except to call the GitHub API directly.
○ Enter credentials above and click Test Connection.

🔄 Sync Files to GitHub

Push all locally stored files to your repository. Already-synced files are skipped automatically. A live progress log shows every upload.

💾 Data Backup & Restore

Your entry data (tests, medicines, bills, expenses, prescriptions) lives in this browser's localStorage. Back it up to GitHub so you can restore it on any device — including your GitHub Pages URL.

☁️
Backup to GitHub
Saves all entry data as data/tracker-data.json in your repo. Run this after adding new entries.
🔄
Restore from GitHub
Loads your saved data from GitHub back into this browser. Use this on a new device or after clearing browser data.
📂 Or import a JSON file
💡 How to use on GitHub Pages: On your original device click Save to GitHub. Then on any other device open the Pages URL → go to 🐙 GitHub Sync → enter your token → click Load from GitHub. Everything restores instantly.
